# 기술자료 작성 가이드

이 폴더는 기술 학습자료를 작성할 때 에이전트가 따라야 하는 작업 절차, 최종 산출물 규칙, Markdown 작성 기준, 내부 작업 산출물 템플릿을 보관한다.

단계별 산출물은 사용자가 읽는 최종 학습자료가 아니라 다음 단계의 입력으로 사용하는 내부 작업 문서다. 따라서 단순 메모가 아니라 의사결정, 근거, 검증 상태를 남기는 작업 문서로 작성한다.

## 파일 순서

0. `AGENTS.md`: 에이전트가 가장 먼저 읽어야 하는 진입 지침
1. `PROCESS.md`: 에이전트가 따라야 할 작업 순서, 반복 조건, 완료 조건
2. `OUTPUTS.md`: 사용자가 읽거나 배포할 최종 산출물 형식과 파일 구조 규칙
3. `MARKDOWN_STYLE.md`: 기준 원고 Markdown 작성 규칙
4. `01_scope.md`: 독자 상태, 목표, 범위, 최종 산출물 형식, 성공 기준 정의
5. `02_research.md`: 조사 자료, 정의, 수식, 알고리즘, 출처 정리
6. `03_outline.md`: 학습 흐름과 장 구조 설계
7. `04_draft.md`: 전체 내용을 갖춘 초고
8. `05_review.md`: 기술 정확성 검증 기록
9. `06_revision.md`: 구조 퇴고와 학습성 개선 계획 및 반영 내역
10. `07_final.md`: 최종 산출물을 만들기 위한 출판 전 Markdown 기준 원고
11. `08_publish.md`: 최종 산출물 구성과 출판 변환 검수 기록

## 공통 규칙

- 가이드 파일과 내부 작업 산출물은 Markdown으로 작성한다.
- 기술자료의 기준 원고는 Markdown이다.
- 최종 산출물 형식은 `OUTPUTS.md`를 따른다.
- 챕터형 Markdown은 기본적으로 MkDocs 자동 탐색에 맞는 `docs/<book-slug>/NN-<chapter-slug>.md` 구조를 사용한다.
- PDF, HTML, ePub, PPT/PPTX 등은 Markdown 기준 원고에서 파생되는 출판 산출물로 취급한다.
- 내부 작업 산출물은 기본적으로 `<자료폴더>/_work/` 아래에 둔다.
- 번호가 붙은 내부 단계 파일을 자료 폴더 루트에 직접 두지 않는다.
- 자료 폴더 루트에는 사용자가 읽거나 배포할 최종 산출물이 바로 식별되어야 한다.
- 상단 메타데이터의 `status`는 현재 상태에 맞게 갱신한다.
- 불확실한 내용은 삭제하지 말고 `TODO`, `검증 필요`, `출처 필요`로 표시한다.
- 기술 검증과 문장 교정은 분리한다.
- 각 단계의 산출물은 다음 단계에서 실제로 참조할 수 있을 만큼 구체적으로 작성한다.
- 독자 상태가 불명확하면 초고를 작성하지 않는다.
- 최소한 숙련도, 선행지식, 학습 목적은 `01_scope.md`에서 확정한다.
- 최종 산출물 형식이 불명확하면 초고를 작성하지 않는다.

## 상태 값

- `draft`: 아직 검토 전인 작업본
- `reviewed`: 기술 검토 또는 구조 검토를 거친 상태
- `final`: 최종 산출물로 사용할 수 있는 상태
