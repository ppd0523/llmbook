## 3. 공통 검수

- [ ] `01_scope.md`의 최종 산출물 형식과 실제 결과가 일치한다.
- [ ] 자료 폴더 루트에서 최종 산출물이 바로 식별된다.
- [ ] 번호가 붙은 내부 단계 파일이 자료 폴더 루트에 직접 노출되지 않는다.
- [ ] 내부 작업 산출물은 `_work/` 아래에 분리되어 있다.
- [ ] 최종 산출물에 내부 작업 메모가 남아 있지 않다.
- [ ] 최종 산출물에 `TODO`, `검증 필요`, `출처 필요` 표시가 없다.
- [ ] 제목 계층이 유지된다.
- [ ] 목차가 정상 생성된다.
- [ ] 내부 링크가 정상 동작한다.
- [ ] 외부 링크가 정상 동작한다.
- [ ] 코드 블록이 깨지지 않는다.
- [ ] 코드 하이라이트가 적용된다.
- [ ] 수식이 정상 렌더링된다.
- [ ] 그림 경로가 깨지지 않는다.
- [ ] 그림 캡션이 유지된다.
- [ ] 표가 화면 또는 페이지 밖으로 넘치지 않는다.
- [ ] 참고문헌이 유지된다.

## 4. MkDocs 챕터형 Markdown 검수

- [ ] 책이 `docs/<book-slug>/` 아래에 있으며 `<book-slug>`가 소문자 kebab-case다.
- [ ] 사이트 전체 진입점 `docs/index.md`와 책 진입점 `docs/<book-slug>/index.md`가 있다.
- [ ] 챕터 파일이 책 폴더 바로 아래에 있다.
- [ ] 챕터 파일명이 `01-<chapter-slug>.md`, `02-<chapter-slug>.md`처럼 두 자리 번호와 소문자 kebab-case를 사용한다.
- [ ] `chapter.md`처럼 반복되는 일반 파일명을 사용하지 않는다.
- [ ] 파일명 자연 정렬 순서와 실제 읽는 순서가 일치한다.
- [ ] `mkdocs.yml`에 책과 챕터의 수동 `nav` 목록을 중복 작성하지 않았다.
- [ ] 챕터 파일명과 첫 번째 `#` 제목이 같은 주제를 가리킨다.
- [ ] 챕터 간 링크가 상대 경로로 동작한다.
- [ ] 내부 링크가 대상 `.md` 파일을 가리킨다.
- [ ] `assets/<chapter-slug>/` 또는 `assets/common/` 경로가 깨지지 않는다.
- [ ] `_work/`가 게시 결과와 탐색 메뉴에 포함되지 않는다.
- [ ] 각 챕터가 독립적으로 읽을 수 있는 제목과 학습 흐름을 가진다.

## 5. 단일 Markdown 검수

- [ ] 단일 Markdown 파일명이 주제를 드러낸다.
- [ ] 문서 하나 안에 `#` 제목이 하나만 있다.
- [ ] 파일을 단독으로 열어도 학습 목표, 본문, 예제, 요약이 이어진다.
- [ ] 상대 경로 이미지와 링크가 정상 동작한다.

## 6. HTML 검수

- [ ] 모바일 화면에서 읽을 수 있다.
- [ ] 목차와 앵커 링크가 동작한다.
- [ ] 코드 블록이 가로 스크롤 또는 줄바꿈 정책에 맞게 표시된다.
- [ ] 이미지 크기가 화면 폭을 넘지 않는다.
- [ ] 접근성에 문제가 되는 제목 계층 누락이 없다.

## 7. PDF 검수

- [ ] 단일 PDF 파일이 생성되어 있다.
- [ ] 페이지 나눔이 어색하지 않다.
- [ ] 코드 줄바꿈이 읽을 수 있다.
- [ ] 수식이 잘리지 않는다.
- [ ] 그림과 캡션이 분리되지 않는다.
- [ ] 표가 페이지 폭을 넘지 않는다.
- [ ] 참고문헌과 링크가 유지된다.

## 8. ePub 검수

- [ ] 목차가 정상 생성된다.
- [ ] 장 단위 이동이 가능하다.
- [ ] 수식과 코드가 독서 앱에서 읽을 수 있다.
- [ ] 그림 크기가 독서 앱 화면에 맞는다.

## 9. PPT/PPTX 슬라이드 검수

- [ ] 단일 PPT 또는 PPTX 파일이 생성되어 있다.
- [ ] 한 슬라이드에 너무 많은 문장이 들어가지 않는다.
- [ ] 코드와 수식이 발표 화면에서 읽을 수 있다.
- [ ] 본문형 설명을 그대로 축소하지 않았다.
- [ ] 발표 흐름이 학습 목표와 연결된다.
