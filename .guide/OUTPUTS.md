# 최종 산출물 규칙

이 문서는 기술 학습자료 작업에서 사용자가 실제로 읽거나 배포할 최종 산출물의 형식과 파일 구조를 정의한다.

`01_scope.md`부터 `08_publish.md`까지의 단계 파일은 작성, 조사, 검토, 변환 검수를 위한 내부 작업 산출물이다. 내부 작업 산출물만 만들고 작업을 완료한 것으로 판단하지 않는다.

## 1. 용어

- 최종 산출물: 사용자가 실제로 읽거나 배포할 수 있는 학습자료 파일 또는 챕터 폴더 구조.
- 기준 원고: 최종 산출물을 만들기 전에 확정한 Markdown 원고. 기본 경로는 `<자료폴더>/_work/07_final.md`다.
- 내부 작업 산출물: 범위 정의, 조사, 구성, 초고, 검토, 퇴고, 출판 검수 기록. 기본 경로는 `<자료폴더>/_work/`다.
- 출판 변환 산출물: 기준 원고에서 변환한 PDF, HTML, ePub, PPT/PPTX 등.

## 2. 작업 시작 전 확인할 항목

초고 작성 전에 다음 항목을 확인한다.

- 최종 산출물 형식
- 최종 산출물 파일명 또는 폴더 구조
- 챕터 수와 챕터 분리 기준
- PDF, PPT/PPTX 등 변환 산출물이 필요한지 여부
- 사용할 변환 도구 또는 금지할 도구
- 예제 코드, 그림, 표, 수식, 참고문헌 포함 여부

최종 산출물 형식이 명시되지 않으면 사용자에게 먼저 묻는다. 사용자가 에이전트 판단에 맡기면 `01_scope.md`에 가정을 기록하고 진행한다.

## 3. 허용되는 최종 산출물 형식

### 3.1 MkDocs 챕터형 Markdown 학습자료

여러 장으로 읽는 학습자료에 사용한다.

이 저장소에서 여러 장으로 읽는 학습자료는 MkDocs 자동 탐색을 기준으로 구성한다. `<자료폴더>`는 `docs/<book-slug>/`에 해당한다.

기본 구조:

```text
docs/
├─ index.md
└─ <book-slug>/
   ├─ index.md
   ├─ 01-<chapter-slug>.md
   ├─ 02-<chapter-slug>.md
   ├─ assets/
   │  ├─ common/
   │  └─ <chapter-slug>/
   └─ _work/
      ├─ 01_scope.md
      ├─ 02_research.md
      ├─ 03_outline.md
      ├─ 04_draft.md
      ├─ 05_review.md
      ├─ 06_revision.md
      ├─ 07_final.md
      └─ 08_publish.md
```

규칙:

- `docs/` 바로 아래의 폴더 하나를 책 한 권으로 취급한다.
- 책 폴더명 `<book-slug>`는 소문자 영문, 숫자, 하이픈만 사용하는 kebab-case로 작성한다. 공백과 밑줄은 사용하지 않는다.
- 사이트 전체 진입점은 `docs/index.md`, 각 책의 진입점은 `docs/<book-slug>/index.md`로 둔다.
- 챕터 본문은 책 폴더 바로 아래에 Markdown 파일로 두며 챕터별 하위 폴더를 만들지 않는다.
- 챕터 파일명은 `NN-<chapter-slug>.md` 형식을 사용한다. `NN`은 `01`, `02`처럼 두 자리 읽기 순서이고, `<chapter-slug>`는 소문자 kebab-case다.
- `chapter.md`처럼 모든 장에 반복되는 일반 파일명은 사용하지 않는다.
- 파일명은 MkDocs 자동 탐색의 정렬 순서와 생성 URL의 기준이다. 챕터 파일명과 문서의 첫 번째 `#` 제목은 같은 주제를 가리켜야 한다.
- `mkdocs.yml`에 책과 챕터를 수동 `nav` 항목으로 반복 등록하지 않는다. 파일과 폴더 배치로 탐색 구조를 생성한다.
- 챕터별 그림과 다운로드 파일은 `assets/<chapter-slug>/`, 여러 챕터가 공유하는 파일은 `assets/common/`에 둔다.
- Markdown 내부 링크는 생성된 HTML URL이 아니라 대상 `.md` 파일을 가리키는 상대 경로로 작성한다.
- `docs/index.md`는 전체 책 목록, 각 책의 `index.md`는 해당 책의 목차와 학습 경로를 제공하며 내부 작업 메모를 포함하지 않는다.
- `_work/`는 내부 작업 산출물이며 MkDocs 게시 대상과 탐색 메뉴에서 제외한다.
- 단계 파일을 자료 폴더 루트에 직접 두지 않는다.

### 3.2 단일 Markdown 학습자료

한 파일로 읽는 짧은 학습자료에 사용한다.

기본 구조:

```text
<자료폴더>/
├─ <자료명>.md
├─ assets/
└─ _work/
   ├─ 01_scope.md
   └─ ...
```

규칙:

- 최종 파일명은 주제를 드러내는 이름으로 정한다.
- 단일 Markdown 파일은 독립적으로 읽을 수 있어야 한다.
- 내부 검토 표, TODO, 검증 필요 표시는 최종 파일에 남기지 않는다.

### 3.3 단일 PDF 학습자료

배포 또는 인쇄 가능한 단일 파일이 필요한 경우 사용한다.

기본 구조:

```text
<자료폴더>/
├─ <자료명>.pdf
├─ source/
│  └─ <자료명>.md
└─ _work/
   ├─ 01_scope.md
   └─ ...
```

규칙:

- 최종 산출물은 단일 PDF 파일이다.
- PDF 생성에 사용한 Markdown 원고와 변환 설정은 `source/` 또는 `_work/`에 보존한다.
- PDF의 페이지 나눔, 표, 코드 블록, 수식, 그림, 링크를 `08_publish.md`에서 검수한다.

### 3.4 단일 PPT/PPTX 슬라이드

발표 또는 강의 슬라이드가 최종 산출물인 경우 사용한다.

기본 구조:

```text
<자료폴더>/
├─ <자료명>.pptx
├─ source/
│  └─ slides.md
└─ _work/
   ├─ 01_scope.md
   └─ ...
```

규칙:

- 최종 산출물은 단일 PPT 또는 PPTX 파일이다.
- 본문형 설명을 그대로 축소하지 말고 슬라이드 흐름으로 재구성한다.
- 발표 화면에서 코드, 수식, 표가 읽히는지 `08_publish.md`에서 검수한다.

### 3.5 기타 출판 형식

HTML, ePub, 웹 사이트, 노트북 등은 사용자 요구가 있을 때 최종 산출물로 사용할 수 있다. 이 경우 `01_scope.md`에 최종 파일 구조와 검수 기준을 명시한다.

## 4. 내부 작업 산출물 위치

내부 작업 산출물은 기본적으로 다음 위치에 둔다.

```text
<자료폴더>/_work/01_scope.md
<자료폴더>/_work/02_research.md
<자료폴더>/_work/03_outline.md
<자료폴더>/_work/04_draft.md
<자료폴더>/_work/05_review.md
<자료폴더>/_work/06_revision.md
<자료폴더>/_work/07_final.md
<자료폴더>/_work/08_publish.md
```

`_work/` 안의 파일은 최종 학습자료가 아니다. 사용자가 최종 산출물 형식을 별도로 지정하지 않은 경우에도 `_work/` 파일만 생성하고 완료하지 않는다.

## 5. 자료 폴더 루트 규칙

- 자료 폴더 루트에는 사용자가 읽거나 배포할 최종 산출물이 바로 식별되어야 한다.
- 번호가 붙은 내부 단계 파일을 자료 폴더 루트에 직접 두지 않는다.
- MkDocs 책 폴더 루트에 둘 수 있는 것은 `index.md`, `NN-<chapter-slug>.md`, `assets/`, `source/`, `_work/`다.
- `source/`는 PDF, PPT/PPTX 등 출판 변환 산출물의 원본을 보존할 때만 사용한다.

## 6. 완료 기준

학습자료 작업은 다음 조건을 만족할 때만 완료된다.

- `01_scope.md`에 최종 산출물 형식과 경로가 기록되어 있다.
- 사용자가 선택한 최종 산출물 파일 또는 챕터 폴더 구조가 실제로 생성되어 있다.
- 내부 작업 산출물은 `_work/` 아래에 분리되어 있다.
- 최종 산출물에는 내부 작업 메모, `TODO`, `검증 필요`, `출처 필요` 표시가 없다.
- MkDocs 챕터형 Markdown이면 `docs/<book-slug>/index.md`와 `NN-<chapter-slug>.md` 형식의 챕터 파일이 존재한다.
- 단일 PDF이면 PDF 파일이 존재하고 `08_publish.md`에 검수 결과가 기록되어 있다.
- 단일 PPT/PPTX이면 슬라이드 파일이 존재하고 `08_publish.md`에 검수 결과가 기록되어 있다.
